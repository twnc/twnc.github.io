ALGORITHMS

Algorithms can be understood as being a clearly defined and actionable set of steps use to calculate a desired value.

The properties of an algorithm are:

  • Input values / Output values
  • Unambiguity
  • Determinism
  • Finiteness
  • Completeness
  • Correctness

There are however exceptions to this. Some programs compromise one or more of these values for pragmatic reasons. A good example of this are Las Vegas algorithms which don't always finish, but when they do they are always correct.

There are many methods used to depict algorithms, both graphically and textually, and these are often used during the planning process. When not writing directly in a programming language the goal is often to write your code in a less detailed manner so that you can draft it more quickly and get a clear overview.

There are many different ways of doing this but a short list includes:

  1. Graphical Methods
  • Flow Charts
  • Structograms (also known as a Nassi-Schneiderman diagram)
  • UML Diagrams (Unified Modelling Language)
  1. Textual Methods
  • Pseudocode (based on programming languages)
  • Stylised Prose
  • Programming Languages

PROGRAMMING LANGUAGES

A programming language is a language with strict formal syntax and vocabulary used in the formation of programs. It's purpose is to create a method of describing algorithms in such way that allows for humans to understand them while simultaneously allowing for automatic conversion into executable machine code.

There are two main types of code:

  • Source Code : Code written in a programming language
  • Machine Code : Code written in zeros and ones that can be executed by a computer (this is usually converted from source code)

Converting Source Code into Machine Code is a process called compiling and that the program that does this is called a compiler. There are however alternatives to this. The most common being a program called a compiler which interprets and carries out the code bit by bit (often line by line). Some programming languages are best suited for compiling while others work better with interpreters.

When compiling most programs don't have the entire program text translated to machine at once. Commonly used parts of programs (such as I/O functionality) will often be stored in libraries where precompiled versions are kept available. These compiled bits of codes are the joined together by a program called a linker.

An alternative to this is a dynamic link library where the compiled parts are linked only when the program is run.

PROGRAMMING PARADIGMS

Different paradigms often suit different sorts of programming and thus different languages often have very different structures. Some of the following will be listed below:

  • Imperative
    • Procedural
      • C++
  • Declarative
    • Functional
      • Lisp
    • Logic-oriented
      • Prolog

Logic-oriented programming comprises of a set of rules and a list of facts. You can then make requests based on this information. For example a program with the rule When it rains, you get wet and the fact It is raining could be asked if you're wet. And given the rules and the information that it has will answer positively. Obviously this is a very simple example but this is the fundamental principle.

Functional programming is made up of functions that are built up in a mathematical way. Each occurence of a function can be replaced by its result. This means that functions don't have any side effects as there are no variables.

Imperative Programming is made up of a series of instructions. These instructions can save, read and edit variables.

There are many other types of programming such as sequential vs parallel, typed versus non typed, and object-oriented programming.

C++ is classified as an imperative, procedural language. This means that it assigns values to variables and that these variables can be reassigned new values and that it is made up of a list of instructions.

It can also be described as a sequential language which means that it's instructions are carried out in the order that they are written.

And finally C++ is also a strongly typed language. This means that variables have a predetermined set of possible values that allow for different sorts of operations such as int, double or string.
